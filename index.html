01 · FOUNDATIONS

What is Metasploit, really?

The Metasploit Framework (MSF) is an open-source platform, maintained by Rapid7, for developing, testing, and running security-assessment modules against target systems in an authorized engagement. It doesn't replace Nmap — it consumes what Nmap finds. Recon tells you a host is running a specific service and version; Metasploit gives you a searchable, standardized library of modules built to test whether that service is actually vulnerable, plus the plumbing (sessions, payload handlers, a shared datastore) to run those tests consistently. Every engagement generally moves through the four stages below.

02 · GETTING STARTED

Installing the framework

Metasploit ships preinstalled on most pentest-focused Linux distributions. Elsewhere, Rapid7 maintains an official installer and a Homebrew formula.

✅

Verify it worked: run msfconsole -v in your terminal — it should print the installed framework version, then drop you at the msf6 > prompt.

05 · DEEP DIVE

Module types, payloads & encoders

Metasploit organizes everything into module types so the console knows how a piece fits together — an exploit needs a payload, a payload might need an encoder, and post modules need an existing session. Understanding this taxonomy matters more than memorizing any single module.

06 · POST-EXPLOITATION

Meterpreter session commands

Meterpreter is Metasploit's in-memory post-exploitation agent — it never touches disk on the target and gives you a consistent command set regardless of the underlying OS. These are the commands you reach for once a session opens, grouped by what stage of the assessment they support. Click a group to expand it.

07 · ADVANCED CONCEPTS

AV & IDS evasion — the concepts, not the code

Endpoint protection actively fingerprints Metasploit's default payload templates. This section explains why evasion techniques exist and what problem each one solves at a conceptual level — it deliberately stops short of ready-to-run bypass code, which ages fast and is easy to misuse outside a lab.

08 · RULES OF ENGAGEMENT

Legal & ethical ground rules

Running exploit modules against a system you don't own or lack explicit written permission to test is illegal in most jurisdictions — this is true even in a Metasploit lab you spun up yourself if it's reachable from outside your isolated network. Run through this list before every session. Click an item to check it off.
